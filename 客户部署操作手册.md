# 家庭健康管理系统 — 部署操作手册

## 一、电脑端运行（H5浏览器版）

### 第1步：安装 Node.js
- 下载地址：https://nodejs.org （选择 LTS 版本）
- 安装完成后打开 cmd，输入 `node -v` 确认版本号

### 第2步：初始化数据库
1. 打开 MySQL Workbench，连接到本地 MySQL（root账号）
2. 菜单 File → Open SQL Script → 选择 `server/database/init.sql`
3. 点击 ⚡ 执行全部 SQL
4. 左侧 SCHEMAS 出现 `health_management` 数据库即成功

### 第3步：配置后端
打开 `server/.env` 文件，确认以下配置：
```
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=你的MySQL密码
DB_NAME=health_management
```

### 第4步：启动后端
```
cd server
npm install
npm start
```
看到"服务器运行在端口 3000"和"数据库连接成功"即可。

### 第5步：启动前端（推荐用 HBuilderX）
1. 打开 HBuilderX
2. 文件 → 打开目录 → 选择 `client` 文件夹
3. 运行 → 运行到浏览器 → Chrome
4. 浏览器自动打开系统页面

## 二、手机 App 打包注意事项

### ⚠️ 必须修改的配置

#### 1. 修改后端地址（最重要！）
打开 `client/utils/request.js`，修改前两行：
```
// 改为你电脑的局域网IP（手机和电脑必须在同一WiFi下）
const BASE_URL = 'http://192.168.x.x:3000/api'
const SERVER_URL = 'http://192.168.x.x:3000'
```

查看电脑IP方法：打开 cmd → 输入 `ipconfig` → 找到"IPv4 地址"（如 192.168.1.100）

**注意：不能用 localhost，手机无法访问电脑的 localhost！**

#### 2. 修复图标不显示问题 ⭐
**原因：UniApp 原生 App 和微信小程序不支持 SVG 图标，只支持 PNG 格式。**

解决方法：
1. 将 `client/static/tabbar/` 下的 8 个 SVG 文件转换为 PNG 格式（建议尺寸 81×81 像素）
   - 可以用在线工具：https://svgtopng.com
   - 将 home.svg → home.png, home-active.svg → home-active.png 等
2. 打开 `client/pages.json`，将所有 `.svg` 后缀改为 `.png`：
   ```
   "iconPath": "static/tabbar/home.png",
   "selectedIconPath": "static/tabbar/home-active.png",
   ```
   其他4组图标同理修改。

#### 3. 修复定位失败问题 ⭐
**原因：**
- App端：需要开启手机GPS并授权定位权限
- H5端：定位需要 HTTPS（HTTP下浏览器会禁止定位）
- 微信小程序：需要配置定位权限

解决方法：
1. **App端**：确保手机设置中打开了GPS定位，App首次使用时弹出的权限请求要点"允许"
2. **H5端**：如果用 HTTP 访问，定位功能不可用，系统会自动降级为 IP 定位
3. **manifest.json 检查**：确认 App 权限配置中包含以下权限：
   - `android.permission.ACCESS_FINE_LOCATION`
   - `android.permission.ACCESS_COARSE_LOCATION`
4. **高德地图 Key**：确认 `server/.env` 中的 `AMAP_KEY` 已配置有效的高德地图 API Key
   - 申请地址：https://lbs.amap.com
   - 创建应用 → 添加 Key → 选择"Web服务"

#### 4. HBuilderX 云打包步骤
1. 用 HBuilderX 打开 `client` 目录
2. manifest.json → 基础配置 → 点击"重新获取" AppID
3. 发行 → 原生App-云打包
4. Android包名填 `com.health.family`
5. 选择"使用公共测试证书"
6. 点击打包，等待 5-15 分钟
7. 下载 APK 传到手机安装

## 三、微信小程序发行注意事项

### 前置条件
- 注册微信小程序账号：https://mp.weixin.qq.com
- 获取小程序 AppID

### 操作步骤

#### 1. 配置 AppID
打开 `client/manifest.json`，在"微信小程序配置"中填入你的 AppID：
```json
"mp-weixin": {
    "appid": "你的真实AppID"
}
```

#### 2. 修改后端地址
- 微信小程序要求后端必须使用 **HTTPS**（不支持 HTTP）
- 后端需要部署到有域名和 SSL 证书的服务器上
- `request.js` 中改为：`https://你的域名/api`

#### 3. 配置合法域名
登录微信公众平台 → 开发 → 开发设置 → 服务器域名：
- request 合法域名：`https://你的域名`
- uploadFile 合法域名：`https://你的域名`

**开发阶段**可在微信开发者工具中勾选"不校验合法域名"跳过此限制。

#### 4. 修复图标不显示（同 App 端）
将 SVG 图标转换为 PNG，修改 `pages.json` 中的路径。

#### 5. 修复定位失败
在 `manifest.json` 的微信小程序配置中添加：
```json
"mp-weixin": {
    "permission": {
        "scope.userLocation": {
            "desc": "您的位置信息将用于查找附近医院"
        }
    }
}
```

#### 6. 构建并上传
1. HBuilderX → 发行 → 小程序-微信
2. 打开微信开发者工具，导入 `client/dist/dev/mp-weixin` 目录
3. 点击"上传"提交到微信后台
4. 登录微信公众平台 → 版本管理 → 提交审核

## 四、常见问题排查

| 问题 | 原因 | 解决 |
|------|------|------|
| 后端启动报"数据库连接失败" | MySQL未启动或密码错误 | 启动MySQL服务，检查.env密码 |
| 页面空白/接口报错 | 后端未启动或地址配错 | 确认后端在运行，检查request.js |
| App图标不显示 | SVG格式不支持 | 转为PNG格式，改pages.json |
| 定位失败 | 权限未授权/HTTP不支持 | 开GPS、授权、用HTTPS |
| 小程序请求失败 | 域名未配置 | 微信后台配合法域名或开发时跳过 |
| 头像/图片不显示 | SERVER_URL配错 | 检查request.js中SERVER_URL |
| AI健康建议不显示 | DeepSeek Key未配 | 在.env配DEEPSEEK_API_KEY |
