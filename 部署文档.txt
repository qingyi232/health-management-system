一、系统概述
================================================================================

    本系统是一个基于 UniApp + Node.js 的家庭健康管理平台，支持 H5、微信小程序、
    Android/iOS App 多端运行。主要功能包括：健康数据记录、健康档案管理、数据分析
    与预警、健康知识推送、就医辅助（医院查询/挂号）、家庭共享等。

    技术栈：
    - 前端: UniApp (Vue 3) + Pinia + Vite + Sass
    - 后端: Node.js + Express + MySQL + Redis
    - 地图: 高德地图 JS API v1.4.15 (含IP定位/逆地理编码)
    - AI:   DeepSeek 大语言模型 (个性化健康建议)
    - 设备: 蓝牙低功耗 BLE (智能手表/手环数据采集)
    - 文件上传: Multer
    - 认证: JWT (JSON Web Token)


二、环境要求
================================================================================

2.1 服务器环境要求
--------------------------
    操作系统:    Windows Server / Linux (推荐 Ubuntu 20.04+)
    Node.js:     v16.0.0 及以上 (推荐 v18 LTS)
    MySQL:       v5.7 及以上 (推荐 v8.0)
    Redis:       v6.0 及以上 (可选，用于缓存)
    磁盘空间:    至少 1GB (含上传文件存储)
    内存:        至少 1GB

2.2 开发环境要求
--------------------------
    Node.js:     v16.0.0 及以上
    npm:         v8.0.0 及以上
    MySQL:       v5.7 及以上
    HBuilderX:   v4.0 及以上 (用于前端开发和真机调试)
    IDE(可选):   VS Code / WebStorm

2.3 端口使用
--------------------------
    后端服务:    3000 (可通过 .env 修改)
    前端H5:      8080 (开发模式，可通过 manifest.json 修改)
    MySQL:       3306 (默认)
    Redis:       6379 (默认)

四、数据库部署
================================================================================

4.1 安装 MySQL
--------------------------
    Windows:
        1. 下载 MySQL Installer: https://dev.mysql.com/downloads/installer/
        2. 选择 "Developer Default" 安装
        3. 设置 root 密码（本项目默认密码为 root，生产环境请修改）
        4. 完成安装后确认 MySQL 服务已启动

    Linux (Ubuntu):
        sudo apt update
        sudo apt install mysql-server
        sudo mysql_secure_installation
        sudo systemctl start mysql
        sudo systemctl enable mysql

4.2 创建数据库并初始化
--------------------------
    方法一：命令行导入
        
        # 登录 MySQL
        mysql -u root -p

        # 执行初始化脚本（在 MySQL 命令行中）
        source /path/to/健康管理系统/server/database/init.sql;

    方法二：使用可视化工具
        
        1. 打开 Navicat / MySQL Workbench / phpMyAdmin 等工具
        2. 连接到 MySQL 服务器 (host: localhost, port: 3306, user: root)
        3. 打开文件: server/database/init.sql
        4. 执行全部 SQL 语句

4.3 数据库说明
--------------------------
    数据库名称: health_management

    排序规则:   utf8mb4_unicode_ci

    │                                    # DeepSeek API 地址       │

    * DEEPSEEK_API_KEY — 替换为真实的 DeepSeek API 密钥
                         (访问 https://platform.deepseek.com 申请)
                         未配置时系统自动降级为本地规则推荐算法

6.5 App 模式打包 (Android APK) — HBuilderX 云打包详细教程
--------------------------

    ⚠️ 注意：HBuilderX 目前仅支持云打包方式生成 APK，不支持本地离线打包。

    【前置条件】
    - 已安装 HBuilderX v4.0 以上版本（官网：https://www.dcloud.io/hbuilderx.html）
    - 已注册 DCloud 开发者账号（https://dev.dcloud.net.cn）
    - 后端服务已部署到可访问的服务器（不能用 localhost）
    - 需要联网（云打包需要上传代码到 DCloud 服务器编译）

    ========================================
    第一步：修改后端接口地址（重要！）
    ========================================
    
    App 不走开发代理，必须使用真实 IP/域名。
    
    打开文件: client/utils/request.js
    修改前两行为后端服务器的实际地址:

        // 局域网测试（手机和电脑在同一 WiFi 下）
        const BASE_URL = 'http://192.168.x.x:3000/api'
        const SERVER_URL = 'http://192.168.x.x:3000'

        // 如果后端部署到公网服务器
        const BASE_URL = 'http://你的服务器IP:3000/api'
        const SERVER_URL = 'http://你的服务器IP:3000'

    ⚠️ 查看电脑局域网 IP 的方法:
        Windows: 打开 cmd → 输入 ipconfig → 找到 IPv4 地址
        例如: 192.168.1.100

    ========================================
    第二步：用 HBuilderX 打开项目
    ========================================
    
    1. 打开 HBuilderX
    2. 菜单栏 → 文件 → 打开目录
    3. 选择 client 文件夹（不是整个项目根目录！）
    4. 项目会出现在左侧项目管理器中

    ========================================
    第三步：配置 manifest.json（应用信息）
    ========================================
    
    在 HBuilderX 中双击打开 manifest.json，会看到可视化配置界面：

    【基础配置】
    - 应用名称: 家庭健康管理（已配置好）
    - 应用版本: 1.0.0（已配置好）
    - uni-app应用标识: 点击"重新获取"获取正式 AppID
      （需要登录 DCloud 账号）

    【App图标配置】（可选）
    - 点击"App图标配置"标签
    - 上传一张 1024×1024 的应用图标图片
    - 点击"自动生成所有图标并替换"

    【App启动界面配置】（可选）
    - 点击"App启动界面配置"标签
    - 可上传自定义启动图或使用默认

    【App模块配置】
    - 已配置好所需权限（网络、相机、存储、定位）
    - 无需额外修改

    ========================================
    第四步：编译运行到手机（可选，用于测试）
    ========================================
    
    方法A - 真机USB调试:
        1. 手机开启"开发者模式"和"USB调试"
           （设置 → 关于手机 → 连续点击"版本号"7次 → 返回设置 →
             开发者选项 → 打开USB调试）
        2. 用USB线连接手机和电脑
        3. HBuilderX 菜单 → 运行 → 运行到手机或模拟器
        4. 选择你的手机设备
        5. 等待编译完成，App 会自动安装到手机上

    方法B - 内置浏览器预览:
        1. HBuilderX 菜单 → 运行 → 运行到内置浏览器
        2. 在内置浏览器中预览效果

    ========================================
    第五步：云打包生成 APK
    ========================================
    
    1. HBuilderX 菜单 → 发行 → 原生App-云打包
    
    2. 在弹出的打包界面中配置:
       ┌───────────────────────────────────────────────┐
       │ Android 包名: com.health.family               │
       │ （格式: com.公司名.应用名，可自定义）          │
       │                                               │
       │ 证书选择: ☑ 使用公共测试证书                   │
       │ （仅用于测试和毕设演示，正式发布需自有证书）    │
       │                                               │
       │ ☑ 传统打包（不勾选"快应用"）                   │
       └───────────────────────────────────────────────┘

    3. 点击【打包】按钮

    4. 等待云端编译（通常需要 5-15 分钟）
       - 打包状态会在 HBuilderX 控制台中显示
       - 完成后会提示"打包成功"

    5. 打包完成后，点击控制台中的下载链接
       - 下载得到 .apk 文件（例如: __UNI__HEALTH_20260321.apk）
       - 将 APK 传到手机上安装即可

    ========================================
    第六步：安装 APK 到手机
    ========================================

7.2 Nginx 配置示例
--------------------------
    server {
        listen 80;
        server_name your-domain.com;

        # 前端 H5 静态资源
        location / {
            root /var/www/health-management/client/dist/build/h5;
            try_files $uri $uri/ /index.html;
        }

        # 后端 API 反向代理
        location /api {
            proxy_pass http://127.0.0.1:3000;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }

        # 上传文件静态服务
        location /uploads {
            proxy_pass http://127.0.0.1:3000;
        }

        # 文件上传大小限制
        client_max_body_size 10m;
    }

7.5 数据库安全
--------------------------
    1. 修改 MySQL root 密码为强密码

       CREATE USER 'health_user'@'localhost' IDENTIFIED BY 'StrongPassword123!';
       GRANT SELECT, INSERT, UPDATE, DELETE ON health_management.* TO 'health_user'@'localhost';
       FLUSH PRIVILEGES;

    3. 定期备份数据库:
       mysqldump -u root -p health_management > backup_$(date +%Y%m%d).sql

    4. 设置定时备份 (crontab):
       0 2 * * * mysqldump -u root -pYourPassword health_management > /backup/health_$(date +\%Y\%m\%d).sql

九、常见问题排查
================================================================================

9.1 后端启动失败: "数据库连接失败"
--------------------------
    原因: MySQL 未启动或连接配置错误
    解决:
    1. 确认 MySQL 服务已启动
       Windows: 服务管理器 → 找到 MySQL → 启动
       Linux:   sudo systemctl start mysql
    2. 检查 server/.env 中的数据库配置是否正确
    3. 确认已执行 init.sql 创建了 health_management 数据库
    4. 测试数据库连接:
       mysql -u root -p -h localhost

9.2 前端页面空白 / 接口报错
--------------------------
    原因: 后端未启动或地址配置错误
    解决:
    1. 确认后端已启动并监听端口 3000
    2. 检查 client/utils/request.js 中的 BASE_URL 地址
    3. 真机调试时不能使用 localhost，需使用电脑局域网 IP
    4. 检查浏览器控制台的网络请求是否有 CORS 错误

9.3 头像/图片无法显示
--------------------------
    原因: 文件 URL 未拼接服务器地址
    解决:
    1. 确认 client/utils/request.js 中的 SERVER_URL 配置正确
    2. 确认后端 uploads 目录存在且有读写权限
    3. 确认 Nginx 配置中 /uploads 路径已正确代理

9.4 高德地图不显示
--------------------------
    原因: API Key 无效或安全域名未配置
    解决:
    1. 确认高德地图 Key 有效且未过期
    2. 在高德控制台添加当前域名到白名单
    3. H5 模式需要 HTTPS 才能使用定位功能

9.5 微信小程序请求失败
--------------------------
    原因: 域名未在微信公众平台配置
    解决:
    1. 登录微信公众平台 → 开发 → 开发设置
    2. 在 "服务器域名" 中添加后端域名 (必须 HTTPS)
    3. 开发阶段可在微信开发者工具中勾选 "不校验合法域名"

9.6 文件上传失败
--------------------------
    原因: 文件类型或大小限制
    解决:
    1. 支持的文件类型: JPEG, PNG, GIF, PDF, DOC, DOCX
    2. 单文件最大: 10MB (可在 .env 中修改 MAX_FILE_SIZE)
    3. 确认 uploads 目录有写入权限:
       Linux: chmod 755 server/uploads

9.7 排班数据过期
--------------------------
    原因: init.sql 中的排班数据基于初始化日期生成未来7天
    解决: 重新执行以下 SQL 刷新排班数据:
       DELETE FROM doctor_schedules;
       INSERT INTO doctor_schedules (doctor_id, schedule_date, time_slot, total, remaining, price)
       SELECT d.id, DATE_ADD(CURDATE(), INTERVAL n.n DAY), t.slot, 20, 20,
              CASE d.title WHEN '主任医师' THEN 50.00 WHEN '副主任医师' THEN 30.00 ELSE 15.00 END
       FROM doctors d
       CROSS JOIN (SELECT 0 n UNION SELECT 1 UNION SELECT 2 UNION SELECT 3
                   UNION SELECT 4 UNION SELECT 5 UNION SELECT 6) n
       CROSS JOIN (SELECT 'morning' slot UNION SELECT 'afternoon') t
       WHERE d.status = 1;


十、新增核心功能说明
================================================================================

10.1 AI 智能健康建议 (DeepSeek 大模型)
--------------------------
    功能描述:
        系统集成 DeepSeek 大语言模型，根据用户的健康数据（血压、血糖、心率、
        体重等）自动生成个性化健康建议，涵盖饮食、运动、生活习惯、医疗提醒
        四个维度。

    工作流程:
        1. 前端首页自动调用 /api/knowledge/ai/advice/:memberId
        2. 后端收集该成员的健康档案和最近健康数据
        3. 构造专业健康助手提示词，调用 DeepSeek API
        4. AI 返回结构化 JSON 建议（overall/diet/exercise/lifestyle/medical）
        5. 前端展示 "AI 智能健康建议" 卡片，支持手动刷新

    降级策略:
        - 若 DEEPSEEK_API_KEY 未配置或为 sk-placeholder → 自动使用本地规则算法
        - 若 AI 接口调用失败（超时/网络错误）→ 自动回退本地规则算法
        - 前端 source 字段区分: "ai" = DeepSeek 生成, "local" = 本地算法

    配置步骤:
        1. 访问 https://platform.deepseek.com 注册账号
        2. 创建 API Key（新用户有免费额度）
        3. 将 API Key 填入 server/.env 的 DEEPSEEK_API_KEY 字段
        4. 重启后端服务即可生效

10.2 智能设备蓝牙 BLE 连接
--------------------------
    功能描述:
        支持通过蓝牙低功耗（BLE）协议扫描并连接附近的智能手表/手环设备，
        实时获取心率、血压等健康数据。

    支持的标准 BLE 健康服务:
        - 心率服务:        UUID 0x180D
        - 血压服务:        UUID 0x1810
        - 健康温度服务:    UUID 0x1809
        - 体重秤服务:      UUID 0x181D
        - 血氧服务:        UUID 0x1822

    使用流程:
        1. 设备绑定页 → 选择"蓝牙扫描"模式
        2. 点击"开始扫描附近设备" → 发现附近 BLE 设备
        3. 选择目标设备 → 自动连接并发现服务/特征值
        4. 绑定完成后，设备列表页可通过"同步"按钮直连蓝牙读取数据

    运行环境要求:
        - Android App:  需授予蓝牙和定位权限（Android 12+ 需 BLUETOOTH_SCAN）
        - iOS App:      需在 Info.plist 添加蓝牙使用说明
        - H5 模式:      仅 Chrome 等支持 Web Bluetooth API 的浏览器可用
        - 微信小程序:   需在 app.json 中声明蓝牙权限

    注意事项:
        - 蓝牙连接失败时自动回退至云端同步（模拟数据）
        - 真机调试时蓝牙功能正常；模拟器不支持蓝牙

10.3 多级定位回退机制
--------------------------
    功能描述:
        医院查询页面采用三级定位回退策略，确保在不同运行环境下均能获取
        用户位置：

    定位优先级:
        第一级: GPS 定位 (uni.getLocation / 高德 JS SDK / 浏览器 Geolocation)
        第二级: IP 定位 (后端 /api/hospital/location/ip → 高德 IP 定位 API)
        第三级: 默认坐标 (经纬度 0,0 + 提示用户手动选择)

    适用场景:
        - App 端: GPS 定位优先，精度最高
        - H5 端: 浏览器定位需 HTTPS；无 HTTPS 时自动降级 IP 定位
        - 微信小程序: 使用 uni.getLocation 获取微信授权位置
